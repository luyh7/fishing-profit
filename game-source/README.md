## 更新契约

收到新的游戏源码压缩包时，按以下顺序处理：

1. 将压缩包以收到时的原文件名保存到 `archives/`，不得重命名、覆盖或删除任何已有压缩包。
2. 在临时目录完整解压并确认成功，不直接向 `current/` 覆盖写入。
3. 校验成功后，用新解压源码完整替换 `current/`；不得合并新旧源码，也不得留下按日期或版本命名的解压目录。
4. 解压或校验失败时保留现有 `current/`，不得留下半更新状态。
5. 如果已有同名压缩包，内容相同则复用；内容不同则停止并报告，不覆盖也不改名。
6. 替换完成后更新本文件中的当前来源、哈希和解压文件数，再以新源码为准分析利润网差异。
