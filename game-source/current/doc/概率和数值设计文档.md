# 钓鱼游戏经济系统设计文档

## 1. 问题描述
在钓鱼游戏中，玩家拥有等级为 `x` 的鱼竿，可以选择进入等级为 `n` （`n ≤ x`）的关卡钓鱼。定义 **钓鱼等级** `d = x - n`。  
- 每个关卡 `n` 只能钓到一种鱼，该鱼的基础价格（稀有度 0 的价格）记为 `base_n`。  
- 同一种鱼存在无穷多个稀有度等级 `r = 0,1,2,...`，其价格仅由基础价格和稀有度决定，与关卡、钓鱼等级均无关。  
- 稀有度出现的概率分布仅依赖于钓鱼等级 `d`，且钓鱼等级越高，高稀有度的出现概率越大（类似离散化的正态分布，且分布集中于少数等级）。  

**核心约束**：  
1. 玩家无论选择哪个可用关卡（即任意 `n ≤ x`），钓鱼的**期望收益**应当相等（或近乎相等）。  
2. 较难的关卡（`n` 大）钓到的鱼稀有度较低，但基础价格高；较简单的关卡（`n` 小）钓到的鱼稀有度较高，但基础价格低。  
3. 不同稀有度的价格通过一个函数 `f(base_n, r)` 由基础价格派生，与钓鱼等级无关。

## 2. 符号体系

| 符号 | 含义 |
|------|------|
| `x`  | 鱼竿等级 |
| `n`  | 关卡等级（`n ≤ x`） |
| `d = x - n` | 钓鱼等级 |
| `base_n` | 第 `n` 关鱼的**基础价格**（即稀有度 0 的价格） |
| `r`  | 稀有度等级，`r ∈ {0,1,2,...}` |
| `P(r ∣ d)` | 钓鱼等级为 `d` 时获得稀有度 `r` 的概率 |
| `C`  | 稀有度价格倍率（全局常数） |
| `M(d)` | 期望价格倍率，`M(d) = E[C^r ∣ d]` |

## 3. 价格函数设计
为满足“价格只取决于基础价格和稀有度”的要求，采用指数价格函数：
\[
\text{price}(base_n, r) = base_n \cdot C^{\,r}
\]
其中 `C > 1` 为全局常数（本例中取 `C = 2`，即稀有度每提高一级价格翻倍）。  
该函数满足 `price(base_n, 0) = base_n`，即稀有度 0 的价格为基础价格。  
初始基础价格设定：关卡 1 的基础价格设为 10（即 `base_1 = 10`）。

## 4. 稀有度分布设计

### 4.1 基本形状
要求分布集中于少数几个等级，且随着钓鱼等级 `d` 的增加，分布的峰值缓慢向右移动。为此采用**离散化的高斯分布**，其未归一化的权重为：
\[
w(r; d) = \exp\left( -\frac{(r - \mu(d))^2}{2\sigma^2} \right), \quad \mu(d) = s \cdot d
\]
其中：
- `s`：均值移动速度，控制峰值位置随 `d` 的变化快慢（本例 `s = 0.4`，即平均每提升 10 个钓鱼等级，峰值右移 4 个稀有度）；
- `σ`：标准差，控制分布的集中程度（本例 `σ = 0.8`，`σ` 越小分布越集中）。

设稀有度上限 `R_max` 为 `μ_max + 5σ` 向上取整，保证覆盖几乎所有非零权重。归一化得到原始概率分布：
\[
\tilde{P}(r \mid d) = \frac{w(r; d)}{\sum_{k=0}^{R_{\max}} w(k; d)}
\]

### 4.2 尾部裁剪
为了进一步减少无关的微小概率事件，将所有概率低于阈值 `θ = 0.05` 的稀有度等级合并到“距离最近且概率 ≥ θ”的等级上（若距离相等，优先合并到较低稀有度）。合并后重新归一化，得到最终的离散分布 `P(r ∣ d)`。

裁剪后的分布特性：对于任意 `d`，绝大多数概率集中在 2~3 个连续稀有度上，其余稀有度的概率为 0。这样玩家在实际游戏中几乎只会见到少数几种稀有度，符合预期。

### 4.3 移动速度
参数 `s = 0.4` 使得：
- `d = 0` 时，峰值在稀有度 0；
- `d = 10` 时，峰值在稀有度 4；
- `d = 19` 时，峰值在稀有度 7~8。

完美实现了“分布移动缓慢”的需求。

## 5. 期望收益恒定条件

玩家在关卡 `n` 钓鱼的期望收益为：
\[
E_n = base_n \cdot \mathbb{E}[C^r \mid d] = base_n \cdot M(d)
\]
其中 `M(d) = \sum_{r} P(r \mid d) \cdot C^r`。

若要使同一鱼竿等级 `x` 下所有关卡 `n` 的期望收益 `E_n` 均相等，记该共同值为 `V(x)`，则需满足：
\[
base_n \cdot M(x - n) = V(x) \quad \forall n \le x
\]
这是一个函数方程。若 `M(d)` 恰好具有指数形式 `M(d) = K \cdot A^d`（`A > 0`），则可推导出：
- 基础价格序列为 `base_n = B \cdot A^n` （`B` 为常数）；
- 期望收益 `V(x) = B \cdot K \cdot A^x`，仅与鱼竿等级 `x` 有关，与关卡 `n` 无关。

### 5.1 实际情形下的近似处理
由于对高斯分布进行了尾部裁剪，`M(d)` 并不完美地呈现为指数函数，但其对数 `ln M(d)` 与 `d` 近似为线性关系。为此，我们对所有钓鱼等级 `d` 计算出的 `M(d)` 进行线性回归：
\[
\ln M(d) \approx \alpha + \beta \cdot d
\]
得到斜率 `β` 和截距 `α`，并令 `A = e^{β}`。  
然后基础价格序列依然采用指数形式：
\[
base_n = B \cdot A^n
\]
其中系数 `B` 由初始定价要求确定：令 `base_1 = 10`，则 `B = 10 / A`。

这一近似会引入微小的期望收益差异（同一 `x` 内不同关卡的期望收益不完全相等），但通常差异在可接受范围内（例如相对差异小于 2%）。若需严格相等，可以对 `base_n` 进行数值微调，但会略微破坏基础价格的完美指数形式。当前设计选择保留指数形式的简洁性，且微小差异不影响核心体验。

## 6. 参数设定

本系统使用的全局参数如下：

| 参数 | 值 | 说明 |
|------|-----|------|
| `C` | 2.0 | 稀有度价格倍率，每级翻倍 |
| `s` | 0.4 | 均值移动速度 |
| `σ` | 0.8 | 离散化高斯的标准差（控制集中度） |
| `θ` | 0.05 | 尾部裁剪阈值 |
| `base_1` | 10.0 | 关卡 1 的基础价格（稀有度 0 的价格） |

**公式汇总**：  
- 价格函数：`price(n, r) = base_n * 2^r`  
- 基础价格序列：`base_n = B * A^n`，其中 `A = e^{β}`，`B = 10 / A`（`β` 由 `ln M(d)` 的线性回归得出）  
- 稀有度分布：`P(r ∣ d)` 是以 `μ=0.4d` 为中心、`σ=0.8` 的离散高斯分布，经裁剪（阈值 0.05）、合并、归一化后得到的概率质量函数。

**注意**：具体的 `A` 和 `B` 数值取决于分布生成和线性拟合的结果，运行程序后即可获得。初始条件 `base_1 = 10` 保证关卡 1 的鱼最低稀有度价格为 10。

## 7. 数据生成方法

基于上述公式，通过程序可生成两张核心数据表：
1. **价格矩阵**：前 `N` 关（如 10 关）稀有度 0..9 的价格，矩阵元素为 `base_n * C^r`。
2. **稀有度分布表**：钓鱼等级 `d = 0..D_max`（如 19）下各稀有度 `r` 出现的概率 `P(r ∣ d)`（通常展示稀有度 0..15）。

由于分布经过裁剪，大部分概率为 0，概率集中于均值附近的 2~3 个稀有度等级。具体数值需运行附带的 Python 脚本生成，本文档不包含固定数据，以确保灵活性（可随时调整参数并重新生成）。

## 8. 数学本质与实现路径

本系统的设计既有严格的数学推导，也依赖程序数值计算：
- **价格函数**和**基础价格指数形式**源于对期望收益恒定条件的数学求解；  
- **稀有度分布**使用离散高斯公式生成，并执行程序化尾部裁剪；  
- **基础价格的最终参数**（`A` 和 `B`）通过对计算得到的 `M(d)` 进行对数线性回归得出，是数学近似与程序实验的结合。

因此，这套方案是“数学推导 + 数据驱动调参”的产物，既保证了系统的可解释性，又通过程序实现获得了满足感官要求的平滑数据。所有约束均得到满足，且允许微小误差。

## 9. 使用说明

1. 确定游戏所需的参数（`C`、`s`、`σ`、`θ`、`base_1`）。
2. 运行生成脚本计算 `M(d)` 并线性回归得到 `A` 和 `B`。
3. 生成价格矩阵和分布表，直接用于游戏配置。
4. 若期望收益差异超出预期，可微调 `σ` 或采用数值方法逐 `x` 调整 `base_n`，但当前设计已足够大多数情况使用。
