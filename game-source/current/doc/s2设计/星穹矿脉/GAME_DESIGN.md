# S2 · 猫猫挖矿（星穹矿脉）游戏设计

> 版本: **v2.1 大胆指数**（控制台 sim_v8 ACCEPT → 网页）  

> 类型: **外观像钓鱼的温和挂机** · 题材是猫猫挖矿  
> 目标周期: **约 30 天通关**  
> 独立性: **不与钓鱼经济 / 天气 / 鱼饵 / 猫乐园 / 流星鱼联动**

---

## 0. 一句话

猫咪矿工队在一颗可挖穿的星球上挂机挖矿：开始挖 → 看状态 → 结束结算。  
矿物有稀有度，矿店每天最多升 3 项。数字会大胆涨到 **∞**，通关条件是 **挖穿星球并达成二阶无限**（先触达一阶无限一次，二阶 +1）。

## 1. 相对旧版：整套推翻

| 旧版（已废弃） | 新版 |
|----------------|------|
| 指数功率 `Δlog10 S = r·Δt` | **离散挖矿 tick** + 稀有度掉落 |

## 2. 对标钓鱼：只学气质，不抄机制

| 钓鱼 | 挖矿（保留的「温和感」） |
|------|--------------------------|
| 钓鱼 / 收杆 / 钓鱼状态 | **挖矿 / 结束挖矿 / 挖矿状态** |
| 按时间间隔出鱼 | 按时间间隔出 **一次挖掘 tick** |
| N~UR 稀有度与色板 | **同档稀有度与色板**（矿名不同） |
| 鱼店升级鱼竿鱼钩 | **矿店**升级钻头 / 镐速 / 矿车 / 爆破 |
| 天气·鱼饵·UTR·乐园… | **全部不要** |

玩家操作节奏应和钓鱼一样「挂着就行」，而不是点破界、堆键石。

## 3. 定案约束

| 项 | 定案 |
|----|------|
| 指令循环 | 仅 **挖矿 / 结束挖矿 / 挖矿状态** |
| 日升级上限 | **硬顶 3**（矿店消费次数） |
| 里程 | **深度 D → 挖穿星球**（`D ≥ D_core`） |
| 数值胆识 | 矿币 / 批量可到 **一阶无限** `10^308` |
| 通关 | **深度挖穿** 且 **二阶无限 ≥ 1** |
| 二阶无限 | 生涯中 **一阶无限触发 1 次** → 二阶计数 +1 |
| 结算 | tick 少：逐次模拟；tick 多：**分布结算** |
| 联动 | 无钓鱼经济、无 UTR/天气/饵/乐园/流星 |

## 4. 核心循环

```
挖矿（开始会话）
    → 时间流逝，累计 pending ticks
挖矿状态
    → 按分布/逐次 结算到「当前时刻」
    → 展示深度、本段矿物、矿币预览（可落库或仅预览，Demo 直接落库步进）
结束挖矿
    → 结算剩余 ticks，关闭会话，写入背包/矿币/深度
矿店
    → 用矿币升级（受每日 3 次硬顶）
```

网页 Demo 用加速时钟模拟「天」与挂机；正式 QQ 版再接到真实时间戳。

## 6. 一次挖掘 tick

1. 按 **钻头等级** 取稀有度概率向量 `p[r]`  
2. 抽稀有度 `r`（大批量时改为分布，见 §8）  
3. 数量 `q = floor(base_q * cart_mult)`（至少 1）  
4. 矿币 `Δcoins = price[r] * q`  
5. 深度 `Δdepth = depth_per_tick * blast_mult`

无天气修正、无双捕特殊事件、无保底树——保持机制极简。

### 6.1 稀有度表（Demo 定案，可调）

钻头等级 `L = 0..20`，令 `t = clamp(L / 20, 0, 1)`，基础权重：

```
wN   = 0.70 - 0.55*t
wR   = 0.22 - 0.05*t
wSR  = 0.06 + 0.18*t
wSSR = 0.015 + 0.25*t
wUR  = 0.005 + 0.17*t
```

归一化后为 `p[r]`。高钻头明显更易出 SSR/UR，但仍保留 N 垫底。

### 6.2 间隔与数量

```
interval_sec = base_interval / speed_mult     # base_interval Demo=8s
qty_mult     = 2 ^ cart_lv * 1.5 ^ lamp_lv * ...   # 朴实乘法叠加
depth_mult   = 2 ^ blast_lv * 1.5 ^ brace_lv * ...
depth_per_tick = 1.0 * depth_mult
mining_seconds_per_day = 2400                      # 抽象日工时 → tick
```

件数与矿币内部用 **log10** 累加（防溢出），展示科学计数。正式服 `base_interval` 再对齐真实墙钟。

### 6.3 日末升级闸

```
跨天前 if (今日升级次数 < 3 且 存在可负担未满级升级):
    冻结 dayProgress ≈ 0.999
    显示「日末暂停」顶栏 + 矿店高亮
    仍可继续挖矿攒币
    直到 3 次用完 或 全部买不起 → 才放行下一天
手动「次日」同样被拦截，不能跳过升级闸
```

## 7. 里程：挖穿星球

```
D_core = 1e306     # 地心深度；内部用 log10=306 比较
进度条 ≈ depth_log / 306   # 早期也能看见进度
挖穿 ⇔ depth_log ≥ 306
```

深度只涨不降。通关必须 **挖穿**，避免纯堆币无视里程。

## 8. 结算：分布近似（关键）

设本段应结算 tick 数 `n`（由时间差 / interval 得出）。

### 8.1 小样本（`n ≤ N_sim`，Demo `N_sim = 40`）

逐 tick 独立抽样，结果可复现随机种子。

### 8.2 大样本（`n > N_sim`）

不逐条模拟，对每个稀有度用 **正态近似多项分布**：

```
mean_r = n * p[r]
var_r  = n * p[r] * (1 - p[r])
count_r = max(0, round(Normal(mean_r, sqrt(var_r))))
```

然后按比例微调使 `Σ count_r = n`（最大项吞误差）。  
每档矿物件数：

```
ore_r = count_r * q_per_tick
```

矿币与深度：

```
Δcoins = Σ price[r] * ore_r
Δdepth = n * depth_per_tick * blast_mult
```

### 8.3 展示

结算面板给：

- 本段 tick 数、用时  
- 各稀有度件数（科学计数）  
- 矿币增量、深度增量  
- 标注「精确模拟」或「分布结算」

后期 `n` 上万时 UI 仍秒级返回。

## 9. 一阶 / 二阶无限

### 9.1 一阶无限

当矿币（log10）或单次入账后总量满足：

```
coins ≥ 10^308    ⇔   coins_log10 ≥ 308
```

- 展示矿币为 `∞`  
- 若尚未触发过：`hit_inf1 = true`，`inf2 += 1`  
- 矿币 **钳制在 10^308**（不再无界暴涨破坏存储）；继续挖仍涨深度与图鉴统计

### 9.2 二阶无限与通关

```
通关 ⇔ depth ≥ D_core 且 inf2 ≥ 1
```

文案：`一阶无限已见证 · 二阶无限 +1 · 地心打通`。

设计意图：约 30 日节奏内，通过矿车/钻头把产出推到 ∞ 一次，同时爆破把深度推穿；**不是**旧版多轮破界。

## 10. 30 日节奏骨架（Demo 目标）

| 日 | 期望体感 |
|----|----------|
| D1–3 | 只会出 N/R，深度缓慢，学会三指令 + 矿店 |
| D4–10 | 钻头/镐速成型，SR 常见，日升级常打满 3 |
| D11–20 | 矿车抬数量，科学计数成为常态 |
| D21–28 | 冲一阶无限；爆破赶深度 |
| D29–32 | 触达 ∞（inf2=1）且 `depth→D_core` 通关 |

日升级预算：30×3 = 90 次理论上限；通关期望总升级约 **55–75**。

价格示意（Demo）：

```
cost(drill, lv) = 20 * 2.6^lv
cost(pick,  lv) = 15 * 2.4^lv
cost(cart,  lv) = 40 * 3.2^lv
cost(blast, lv) = 25 * 2.9^lv
```

矿车负责把数量送进 ∞（每级 +8 log10）；爆破负责地心。

## 11. 指令规格（正式服预留）

| 指令 | 行为 |
|------|------|
| `挖矿` | 若空闲则开会话；已在挖则提示状态 |

| `升级钻头/镐速/矿车/爆破` | 扣矿币、耗 1 次日升级 |

## 14. 验收标准

1. 不升级时产出稳定、不出现「无操作却指数自加速」的误解（间隔固定，期望线性于时间；数量倍率只来自矿车）。  
2. `n` 很大时结算走分布路径且总和守恒。  
3. 日升级第 4 次被硬拒。  
4. 可在加速模式下约 30 个游戏日内打出通关（参数允许 ±20% 调参）。  
5. 通关时 `inf2 ≥ 1` 且深度满。
