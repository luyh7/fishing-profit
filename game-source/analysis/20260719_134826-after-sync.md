# 20260719_134826 新版逻辑同步后复核

## 结论

利润网已同步新版游戏源码对稀有度概率链的四项修复：高阶原始槽归并到普通表上限、S1 普通表上限改为 UR、流星改为封顶后调整，以及幸运药水只按最终稀有度取优。新版审计报告中由这四项产生的利润差异已经消除。

本轮没有修改用户未点名的遗留差异，也继续保留用户指定口径：展示木框价值为 0，猫猫框价值为 200。新版源码已经取消猫猫框出售，因此 200 现在是利润网的指定估值，不是游戏售价。

本报告以游戏实际结算路径 [`core/engine.py`](../current/core/engine.py) 为对齐口径；修复前差异和完整遗留清单见 [`20260719_134826.md`](./20260719_134826.md)。

## 同步内容

| 新版变化 | 利润网同步结果 | 主要实现 |
| --- | --- | --- |
| 高阶原始槽不再回退 N | 先把超过普通表上限的全部概率质量归并到上限 | [`catch-outcome.js`](../../catch-outcome.js) |
| S1 普通表上限改为 UR | S1 配置改为 UR，普通表不再自然产出 UTR | [`config.js`](../../config.js) |
| 实际流星路径改为先封顶后调整 | 归并后再从次高档向上限移动 2 个百分点 | [`catch-outcome.js`](../../catch-outcome.js) |
| 幸运不再保留原始槽优先级 | 普通鱼只按封顶后的最终稀有度和图鉴状态取优 | [`catch-outcome.js`](../../catch-outcome.js) |

页面概率、当前单鱼期望价值和地图收益共用同一套最终分布，`app.js` 不存在需要单独修改的旧概率分支。

## 源码边界处理

新版实际流星实现会在 `d>=12` 的已封顶分布上继续从空的次高档扣除 2%，再把不足量转移到 N，形成 `N=-2% / UR=102%` 的无效内部数组；实际按累积概率抽样时仍恒定得到 UR。利润网在次高档无可移动概率时保持 UR 100%，与可观察的游戏抽样结果一致，同时避免向页面暴露负概率或超过 100% 的概率。

## 仍保留的差异

修复前报告列出的未点名差异仍保持不变，包括 24 小时尾段判定、特殊地图准入道具、库存与逐时状态、其他药水和 buff、星空鱼额外收益、首条 UTR 消耗、猫礼物选鱼策略、猫猫框保底、非法建筑组合、当前保底计数、水晶猫城堡与幸运的相关性、手动星空成就输入、过山车对迷途风增幅，以及未完成成就时手动选择迷途风的门控差异。本轮没有扩展修复范围。
