# 钓鱼收益计算器需求说明

## 范围

页面分成两块：

- 上半部分保持原有鱼饵收益计算逻辑不变。
- 原始计算参数卡片显示鱼钩、鱼竿和 buff。
- 地图选择放在右侧。

## 原始逻辑

原始鱼饵收益模型保持不变，仍然按以下方式计算：

```text
钓鱼间隔 = 1小时 / (1 + 鱼钩加速) / (1 + 鱼饵加速 + 打窝buff) / (1 + 系统buff)
```

说明：

- 鱼钩等级、鱼竿等级、鱼饵列表、鱼钩加速、鱼饵价格都从配置文件读取。
- 打窝 buff 和系统 buff 继续按百分比输入和显示，例如输入 30 表示 30%。
- 用户输入的参数需要本地保存，页面下次打开时自动恢复。
- 页面仍然显示最优鱼饵、24 小时理论次数、完成次数、总收益、成本和净收益。

## 新增图鉴内容

新增内容只负责展示鱼竿与地图配置，不改变原始鱼饵收益逻辑。

### 鱼钩

- 鱼钩等级从 1 到 10。
- 鱼钩负责加速。

### 鱼竿

- 鱼竿等级从 0 到 10。
- 鱼竿等级决定可选地图上限。

### 地图

- 地图配置来自同一个配置文件。
- 仅显示地图等级小于等于鱼竿等级的地图。
- 默认选中第一个可用地图。
- 地图卡片仅展示该地图的净收益。
- 地图卡片同时以浅色显示该地图的最优鱼饵。

### 稀有度与价格

- 稀有度从高到低为 UR、SSR、SR、R、N。
- 每张地图内配置自己的鱼种列表与各鱼种的 N 级基础价格，不再使用全局 `nFishPrice`。
- 选中地图后，页面先按该地图的鱼种 N 级基础价求平均，再按稀有度倍数推导鱼均价。

## 图鉴计算规则

- 图鉴部分按鱼竿等级减去地图等级得到差值，并读取对应概率表。

页面应显示：

- 当前地图概率、单鱼期望价值、24 小时期望收益。
- 所有可选地图卡片。

## 交互要求

- 任一输入变化后结果应自动刷新。
- 原始鱼饵计算和地图选择联动。
- 原始逻辑不得被替换或删除。
- 所有配置内容集中在同一个配置文件，方便后续调整。

## 同步要求

后续任何涉及公式、鱼竿、地图、概率或显示字段的变更，都必须同步更新本文件。
