# 钓鱼收益计算器需求说明

## 目标

页面提供一个钓鱼收益计算器，基于配置文件中的鱼钩、鱼竿、鱼饵、地图、概率与 buff 数据，计算 24 小时内的完成次数、总收益、成本和净收益，并给出最优鱼饵与最优地图。

## 数据来源

- 所有基础配置集中在 `config.js`。
- 页面直接读取鱼钩等级、鱼竿等级、鱼饵列表、系统 buff、地图列表、稀有度倍数和概率表。
- 每张地图内配置自己的 `id`、`difficulty`、鱼种列表及每种鱼的 N 级基础价格。
- 用户输入与选择会通过 `localStorage` 持久化，包括鱼钩等级、鱼竿等级、系统 buff、当前地图、每张地图的打窝 buff 和自动更新打窝 buff 开关状态。
- 地图打窝 buff 支持通过配置中的静态 JSON 地址刷新，地址由 `config.js` 中的 `nestBuffSourceUrl` 提供，推荐指向 Cloudflare Worker 的 `/nest-buff.json`，返回的 `locations[].buffs.nest` 会按 `nest * 5` 转换为最终打窝 buff。

## 页面结构

- 顶部显示版本信息和参数选择区、按鱼竿等级过滤后的地图卡片。
- 中部显示当前选中地图的摘要信息。
- 下方显示鱼饵收益表。

## 输入项

- 鱼钩等级：从配置中选择，影响钓鱼间隔。
- 鱼竿等级：从配置中选择，决定可查看地图的上限。
- 系统 buff：从配置中选择，当前可选值为无和周末加速 30%。
- 地图打窝 buff：每张地图单独维护，支持按 5% 步进加减，最小值为 0。

## 计算规则

### 1. 钓鱼间隔

```text
钓鱼间隔 = 基础间隔 / (1 + 鱼钩加速) / (1 + 鱼饵加速 + 打窝buff/100) / (1 + 系统buff/100)
```

其中：

- 基础间隔来自配置，当前为 1 小时。
- 打窝 buff 以百分比参与计算，例如 30 表示 30%。
- 系统 buff 也以百分比参与计算。

### 2. 地图单鱼期望价

对当前地图内所有鱼种的 N 级基础价格求平均，再按稀有度概率与倍数计算单鱼期望价：

```text
单鱼期望价 = 平均 N 级基础价 × Σ(各稀有度概率 × 稀有度倍数)
```

迷途风天气下，会先固定 1% 为 UTR，剩余 99% 按原概率分布等比缩放；UTR 倍数为 UR 的 2 倍。

稀有度顺序为 UTR、UR、SSR、SR、R、N，UTR 仅在迷途风天气下出现，倍数由配置提供。

### 3. 鱼饵收益

对每个鱼饵分别计算：

- 24 小时理论次数 = 24 小时 / 钓鱼间隔
- 24 小时完成次数 = 24 小时理论次数向下取整
- 总收益 = 完成次数 × 单鱼期望价
- 成本 = 完成次数 × 鱼饵价格
- 净收益 = 总收益 - 成本

页面会在当前选中地图下选出净收益最高的鱼饵作为最优鱼饵；若净收益相同，取鱼饵 ID 更小者。

## 地图规则

- 仅显示地图等级小于等于鱼竿等级的地图。
- 默认优先选中本地保存的地图；如果不存在或不可用，则选中当前可见列表中的第一个地图。
- 地图卡片左上角序号使用地图 `id`，地图逻辑筛选与差值计算使用 `difficulty`。
- 地图卡片展示地图等级、名称、该地图下的最优鱼饵、最优鱼饵对应的净收益，以及打窝 buff 调整器。
- 如果某张地图存在任意鱼种的 N 级基础价格为 0，或者该地图对应的 `probabilityByDelta` 概率全部为 0，则该地图不可选中，卡片显示“暂无数据，请等待作者更新”，并且不参与最优地图与当前选中地图计算。
- 地图选择区域标题右侧提供“自动更新打窝buff”开关；开启后会拉取配置中的最新地图打窝静态数据，并每 1 分钟自动同步一次页面计算结果。如果最近 1 分钟内已经成功获取过数据，重新开启开关时不会立刻重复获取，而是在距离上次成功获取满 1 分钟后继续自动更新。开关状态会持久化到本地，下次打开页面时继承；用户手动调整任意地图的打窝 buff 后，开关自动关闭并清除上一次更新标记与更新时间提示。刷新成功后会展示 `update_at` 对应的更新时间，格式为 `yyyy-MM-dd HH:mm:ss`。更新时间显示在报错信息左侧，窄屏时自动换行；报错信息独立展示，不覆盖更新时间文本。
- 地图卡片上的最优标记由当前所有可见地图中净收益最高者决定。

## 当前选中地图展示

选中地图后，摘要区显示：

- 地图名称与等级。
- 渔力差值，即鱼竿等级减去地图等级。
- 鱼钩加速。
- 当前系统 buff。
- 当前地图的打窝 buff。
- 当前地图的稀有度概率分布会根据天气调整，仅显示概率大于 0 的稀有度；迷途风下会额外显示 UTR。
- 当前地图的单鱼期望价。
- 当前地图下最优鱼饵的 24 小时完成次数与净收益。

## 鱼饵表

- 表格按鱼饵 ID 升序展示。
- 每行显示鱼饵名称、价格、加速、钓鱼间隔、24 小时理论次数、24 小时完成次数、总收益、成本和净收益。
- 当前最优鱼饵行高亮显示。
- 当没有可计算结果时显示空状态。

## 其他展示

- 选中地图时会显示该地图下各鱼种在不同稀有度下的单鱼价格提示表。
- 页面标题与版本徽标由配置中的版本前缀和提交计数拼接生成。

## 交互要求

- 任一输入变化后，页面立即重算并刷新显示。
- 切换地图时刷新摘要、地图高亮和鱼饵表。
- 调整地图打窝 buff 时，仅影响对应地图的收益计算，不影响其他地图。

## 同步要求

后续任何涉及公式、鱼钩、鱼竿、地图、概率、buff、收益字段或展示逻辑的变更，都必须同步更新本文件。

## Cloudflare Worker sync

Nest buff data is served by the Cloudflare Worker in `workers/nest-buff-sync`.
Because the upstream IP blocks Cloudflare Worker egress, refresh is pushed by an
external sync runner instead of a Worker cron trigger. The Tencent Cloud
Function fetches the upstream JSON and writes it directly to Cloudflare KV with
the Cloudflare API token. The page reads `/nest-buff.json` through `config.js`
`nestBuffSourceUrl`.
