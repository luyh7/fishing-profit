# 钓鱼收益计算器需求说明

## 目标

页面提供一个钓鱼收益计算器，基于配置文件中的鱼钩、鱼竿、鱼饵、地图、概率与 buff 数据，计算 24 小时内的完成次数、总收益、成本和净收益，并给出最优鱼饵与最优地图。

## 数据来源

- 所有基础配置集中在 `config.js`。
- 页面直接读取鱼钩等级、鱼竿等级、鱼饵列表、周末buff、药水列表、地图列表、稀有度倍数和概率表；当前配置包含普通地图 1-20 和猫猫乐园 `S1`。
- 每张地图内配置自己的 `id`、`difficulty`、鱼种列表及每种鱼的 N 级基础价格；`id` 可为数字或字符串，页面展示、地图选择、打窝同步和成就匹配均按 `id` 识别。
- 用户输入与选择会通过 `localStorage` 持久化，包括鱼钩等级、鱼竿等级、周末buff、药水、当前地图、每张地图的打窝 buff 和自动更新打窝 buff 开关状态。
- 地图打窝 buff 支持通过配置中的静态 JSON 地址刷新，地址由 `config.js` 中的 `nestBuffSourceUrl` 提供，推荐指向 Cloudflare Worker 的 `/nest-buff.json`，返回的 `locations[].buffs.nest`、`locations[].buffs.frame` 和 `locations[].buffs.starry_bonus` 会按 `(nest + frame + starry_bonus) * 5` 转换为最终打窝 buff。
- 静态 JSON 的 `players` 数据用于 QQ 号匹配；匹配成功后可同步玩家昵称、鱼竿等级、鱼钩等级、药水、猫猫乐园建筑等级、当前位置、当前鱼饵及数量和 `collect_scene_*` 场景成就。
- 静态 JSON 的 `players[].buffs` 用于自动药水识别。当前识别 `type` 为 `duoduo` 或 `lucky_double`、`value` 大于 0 且满足 `start_time <= 当前时间 < end_time` 的药水；多个有效条目取结束时间最晚的一条，未知类型、非法时间、未开始和已结束条目均不生效。
- 静态 JSON 的 `players[].collections` 用于图鉴读取；普通地图 1-20 按页面地图顺序读取，猫猫乐园固定读取数组下标 20。数组中每个字符串对应一个地图的鱼收集位图，当前使用 6-bit 可打印字符编码：每个字符的 `charCode - 32` 表示 6 个低位优先的 bit 分块。解码后的位图按鱼种顺序分组、每组 6 位，位序从低到高依次为 N、R、SR、SSR、UR、UTR，`bit0` 表示 N，`bit5` 表示 UTR。旧版十进制数字字符串仍兼容读取。

## 页面结构

- 顶部显示版本信息和参数选择区、按鱼竿等级过滤后的地图卡片。
- 中部显示当前选中地图的摘要信息。
- 下方显示鱼饵收益表。

## 输入项

- 鱼钩等级：从配置中选择，当前配置开放 Lv.0-Lv.10，影响钓鱼间隔。
- 鱼竿等级：从配置中选择，当前配置开放 Lv.0-Lv.21；自动更新数据返回的 `players[].rod_level` 已包含传奇猫雕像效果，页面不再额外叠加鱼竿等级。手动模式下，用户需自行选择包含该效果的实际鱼竿等级。
- 周末buff：从配置中选择，当前可选值为无和加速30%。
- 药水：从配置中选择，当前可选值为无、多多药水和幸运药水。自动更新数据开启并匹配到 QQ 玩家时，根据 `players[].buffs` 自动选择；自动模式下悬停药水区域显示已持续时间和剩余时间，手动模式选择有效药水时显示药水效果和“手动模式下不会结束”。
- QQ 号：填写后，在自动更新数据开启时匹配 `players[].user_id`，同步鱼竿等级、鱼钩等级、猫猫乐园建筑等级及玩家状态；点击输入框包裹区域或昵称附近区域应聚焦 QQ 输入框。
- 图鉴：在 QQ 号成功匹配玩家数据后显示图鉴按钮；图鉴内容仅用于展示同步到的 `players[].collections`，页面内不允许点击修改。
- 玩家当前鱼饵：自动模式下，如果玩家当前位置对应地图的最优鱼饵 ID 与玩家当前鱼饵 ID 不一致，玩家当前鱼饵面板背景变为红色。
- 排行榜：当 QQ 号成功匹配玩家数据后，排行榜入口和弹窗摘要显示当前榜单下的“我的排名”，只展示名次，不展示总人数或我的等级；排行榜条目不展示昵称后的详情小字。
- 地图打窝 buff：每张地图单独维护，支持按 5% 步进加减，最小值为 0。
- 鱼饵提醒：当匹配到玩家当前鱼饵数量后，用户可设置提醒阈值并开启提醒；当前数量低于阈值时显示页面中央弹窗提示，弹窗外内容模糊化，用户点击“知道了”后关闭；重复提醒间隔为 10 分钟。

## 计算规则

### 1. 钓鱼间隔

```text
钓鱼间隔 = 基础间隔 / (1 + 鱼钩加速) / (1 + 鱼饵加速 + 打窝buff/100) / (1 + 周末buff/100)
```

其中：

- 基础间隔来自配置，当前为 1 小时。
- 打窝 buff 以百分比参与计算，例如 30 表示 30%。
- 周末buff 也以百分比参与计算。

### 2. 药水

多多药水生效时，计算使用的有效鱼竿等级降低 1 级，最低为 Lv.0：

```text
有效鱼竿等级 = max(0, 选择的鱼竿等级 - 药水等级惩罚)
```

多多药水会将每次鱼获的基础数量设为 2。该数量只作用于鱼获部分，不放大猫猫乐园材料或展示木框收益，也不改变钓鱼间隔和完成次数；与猫猫乐园旋转逗猫棒的额外一条鱼概率按 `基础数量 + 额外概率` 组合，例如 Lv.3 时为 `2 + 10% = 2.1` 条。当前单鱼期望价值仍展示未计入数量倍率的鱼价、材料价值和展示木框价值，数量倍率只进入收入、总收益和净收益计算。

幸运药水不会改变鱼竿等级、钓鱼间隔、完成次数或鱼饵消耗。未触发保底时，每次完成钓鱼会独立判定 2 次结果并按 `展示木框 > 特殊 UTR > 普通鱼封顶后的稀有度` 取优，不比较鱼价；同为普通鱼且稀有度相同时，有图鉴数据则优先未收集鱼，否则保留第一次结果。猫猫乐园建筑未全部达到 Lv3 时，材料优先级高于所有非材料结果；9 栋建筑全部达到 Lv3 后，所有非材料结果优先于材料。水晶猫城堡的鱼竿等级 +1 在每次捕获开始时只判定一次，幸运药水的两个随机结果共享该鱼竿等级；展示木框和特殊 UTR 保底在幸运判定之前检查，两次随机判定取优后只更新一次共享保底计数；猫天气与鱼获数量等后续效果作用于最终结果。

### 3. 地图单鱼期望价

对当前地图内所有鱼种的 N 级基础价格求平均，再按稀有度概率与倍数计算单鱼期望价：

```text
单鱼期望价 = 平均 N 级基础价 × Σ(各稀有度概率 × 稀有度倍数)
```

幸运药水生效时不再直接使用地图平均 N 级基础价，而是展开每个“鱼种 × 稀有度”和材料的完整结果分布，对两次独立判定择优后再汇总鱼价、稀有度概率与材料概率。

稀有度概率按渔力差值查 `rarityDistribution` 的 20 行、每行 16 个原始概率槽，差值钳制到 0-19。实际结算先把高于普通概率表上限的所有槽位归并到上限，再应用流星天气并抽样；1-10 图与 S1 的普通表上限为 UR，11-20 星空图的普通表也固定封顶到 UR，UTR 只通过独立的特殊掉落产生。流星最多把 2 个百分点从上限的相邻次高档移到上限，次高档不足时不产生负概率或超过 100% 的概率。

1-10 图和 S1 在已解锁的迷途风天气下、11-20 星空图在玩家完成当前图 `collect_scene_<id>` 成就后，特殊 UTR 基础概率按有效鱼竿等级和场景等级计算：

```text
场景等级 = difficulty + 1
UTR概率 = 0.2% + max(0, 有效鱼竿等级 - 场景等级) × 0.1%
```

非星空图在材料判定后依次判定 0.7% 展示木框和特殊 UTR；展示木框、特殊 UTR 均有 150 次保底，展示木框保底优先。收益计算在幸运取优后用两个保底计数的长期稳态分布折算结果概率，并按鱼获实际数量推进计数。展示木框替代该次鱼获，价值按 0 计，但掉落率需要显示在当前单鱼期望价值面板；UTR 倍数为 UR 的 2 倍。猫猫乐园中，基础鱼竿等级与城堡 +1 分支分别完成幸运取优，再按城堡触发概率混合，最后统一应用共享保底；过山车天气增幅会把迷途风特殊 UTR 原始概率乘以 `1 + 天气增幅百分比`。

猫天气下，不改变稀有度概率分布，但每条钓起的鱼有 15% 概率被猫吃掉。被吃掉后按以下概率触发收益：30% 提供未计猫咖加成的原鱼售价整数除以 2 的金币，15% 提供猫猫框（按 200 价值计），15% 提供玩家当前装备鱼饵 3 个，30% 提供香甜玉米 1 个（收益计 0），10% 提供同稀有度鱼奖励：如果玩家已完成当前地图成就，则提供当前场景下相同稀有度的最高价格鱼；否则按当前页面口径使用同稀有度随机鱼的平均价值。每条被吃掉的鱼推进一次猫框保底计数，连续 15 次未获得猫框时强制获得并重置；强制保底会替代该次原随机礼物。幸运药水生效时，如果第一次礼物不是猫猫框，会额外做一次独立的 15% 猫框判定，计入 15 次保底后的长期猫框率约为 27.96338%，普通模式长期猫框率约为 16.43573%。鱼饵收益只按玩家数据中的 `bait_id` 匹配配置价格；没有匹配到鱼饵时，按当前地图最优鱼饵价格计算。

猫天气对用户展示的效果文案为“哈基米把你的鱼吃掉了！”，天气浮窗需补充轻快说明：哈基米会在吃掉鱼时送出价值不可估量的礼物；该说明使用与浮窗标题一致的字体样式，以便和时间信息区分。

```text
猫天气单鱼期望价 = 原鱼价期望 × (1 - 吃鱼率) + 吃鱼率 × (金币礼物期望 + 猫框期望 + 鱼饵礼物期望 + 同稀有度鱼礼物期望)
```

当地图存在材料或展示木框掉落时，猫天气只作用于真正的鱼获。一次多条鱼获会逐条判定是否被吃；只有整组鱼全部被吃时该次不消耗鱼饵，材料和展示木框仍会消耗鱼饵。

暴雨天气已开始且尚未结束时，天气浮窗需显示两行：`预计还需xx个xx饵` 和 `消耗速率xx个/h`。计算时使用当前地图最优鱼饵的钓鱼间隔和暴雨的鱼饵消耗倍率，剩余鱼饵数量向上取整为整数。

稀有度顺序为 UTR、UR、SSR、SR、R、N。UTR 仅来自 1-10 图和 S1 已解锁迷途风时的特殊掉落，或 11-20 图完成场景成就后的常驻特殊掉落；倍数由配置提供。

### 4. 鱼饵收益

对每个鱼饵分别计算：

- 24 小时理论次数 = 24 小时 / 钓鱼间隔
- 1 小时理论收入 = 单次收益期望 / 钓鱼间隔
- 24 小时完成次数 = 24 小时理论次数向下取整
- 总收益 = 完成次数 × 单次收益期望；单次收益期望会计入幸运药水的两次判定取优、展示木框和共享保底，以及 `药水基础数量 + 猫猫乐园额外一条鱼概率`，但材料和展示木框价值只加一次
- 成本 = 完成次数 × 鱼饵价格
- 净收益 = 总收益 - 成本

页面会在当前选中地图下选出净收益最高的鱼饵作为最优鱼饵；若净收益相同，取鱼饵 ID 更小者。

## 地图规则

- 地图准入按玩家原始鱼竿等级过滤；多多药水的鱼竿等级 -1 不会关闭玩家已能进入的地图，只影响渔力差值、普通稀有度概率和特殊 UTR 概率。
- 默认优先选中本地保存的地图 `id`；如果不存在或不可用，则选中当前可见列表中的第一个地图。
- 地图卡片左上角序号使用地图 `id`，地图逻辑筛选与差值计算使用 `difficulty`。
- 地图卡片展示地图等级、名称、该地图下的最优鱼饵、最优鱼饵对应的净收益，以及打窝 buff 调整器。
- 猫猫乐园是特殊地图，地图卡片提供建筑加成入口；建筑等级可手动设置并持久化，当前开放到 Lv3。自动更新数据开启并匹配到 QQ 玩家时，会从 `players[].cat_park` 按建筑展示顺序同步建筑等级；用户手动调整任意建筑等级后，自动更新数据开关会关闭。建筑加成只作用于猫猫乐园，不影响其他地图。
- 猫猫乐园建筑加成按每级配置累加；Lv3 累计效果为鱼饵节省 10%、钓鱼速度 +10%、材料率 45%、鱼价 +10%、天气增幅 10%、双倍鱼获 10%、钓鱼等级+1 概率 30%、每日签到 3 抽，传奇猫雕像 Lv3 的效果仍展示鱼竿等级 +1 和材料价值 ¥80，但鱼竿等级 +1 不由页面自动应用，并提示“请自行选择实际等级或自动更新”。传奇猫雕像升到目标 LvN 时，需要其他 8 栋建筑全部达到 LvN；其他建筑升到 Lv2/Lv3 时，分别要求传奇猫雕像达到 Lv1/Lv2。建筑等级只允许逐级提升，不提供手动降级；旧本地数据若存在非法组合，会在加载时只向下收敛为可保留的最大合法状态。猫爬架广场 0 级默认提供材料率 30%，Lv1 额外提供 5%，Lv2 再额外提供 5%，Lv3 再额外提供 5%。建筑弹窗的下级效果展示下一级累计效果。材料率会挤压鱼类稀有度概率并影响单鱼期望与利润；每日签到抽数、解锁等级只作为显示，不参与利润计算。
- 猫猫乐园中，钓鱼速度加成是独立乘区；鱼价按每个“鱼种 × 稀有度”完成 `floor(N价 × 稀有度倍率 × (1 + 加成))`，不会影响材料价值；鱼饵节省减少鱼饵成本；双倍鱼获概率与多多药水基础数量相加，不放大材料收益；钓鱼等级+1概率在每次捕获只判定一次，幸运药水两个结果共享该等级；天气增幅按当前生效天气的速度、消耗、流星、迷途风 UTR 或猫天气效果强度放大。
- 当 QQ 号成功匹配玩家图鉴数据后，若玩家在某张地图当前可钓到的最高稀有度鱼尚未全收集，该地图卡片边框使用该稀有度的半透明颜色提示；该提示优先级低于当前选中地图样式。
- 如果某张地图存在任意鱼种的 N 级基础价格为 0，或者该地图经 `rarityDistribution` 映射后的概率全部为 0，则该地图不可选中，卡片显示“暂无数据，请等待作者更新”，并且不参与最优地图与当前选中地图计算。
- 地图选择区域标题右侧与猫猫乐园建筑弹窗累计效果右侧提供“自动更新打窝buff”开关；开启后会拉取配置中的最新地图打窝静态数据，并每 1 分钟自动同步一次页面计算结果。如果最近 1 分钟内已经成功获取过数据，重新开启开关时不会立刻重复获取，而是在距离上次成功获取满 1 分钟后继续自动更新。开关状态会持久化到本地，下次打开页面时继承；用户手动调整任意地图的打窝 buff、天气、周末buff、药水或猫猫乐园建筑等级后，开关自动关闭并清除上一次更新标记与更新时间提示。刷新成功后会展示 `update_at` 对应的更新时间，格式为 `yyyy-MM-dd HH:mm:ss`。更新时间显示在报错信息左侧，窄屏时自动换行；报错信息独立展示，不覆盖更新时间文本。自动模式下，当天气或玩家药水跨过开始、结束时间边界时，页面需要立即重新计算并刷新地图利润、最佳鱼饵和当前地图摘要，不等待下一次 1 分钟数据同步。
- 地图卡片上的最优标记由当前所有可见地图中净收益最高者决定。

## 当前选中地图展示

选中地图后，摘要区显示：

- 地图名称与地图 `id` 标签，不显示等级前缀。
- 渔力差值，即有效鱼竿等级减去地图等级。
- 鱼钩加速。
- 当前周末buff。
- 当前药水。
- 当前地图的打窝 buff。
- 当前地图的稀有度概率分布会根据天气、药水、材料、展示木框和保底调整，仅显示概率大于 0 的稀有度；摘要额外显示展示木框的长期掉落率，猫天气不改变稀有度概率但会调整单鱼期望价。
- 当前地图的单鱼期望价；展示木框价值固定按 0 计。
- 当前地图下最优鱼饵的 1 小时理论收入与净收益。

## 鱼饵表

- 表格按鱼饵 ID 升序展示。
- 每行显示鱼饵名称、价格、加速、钓鱼间隔、24 小时理论次数、1 小时理论收入、总收益、成本和净收益。
- 当前最优鱼饵行高亮显示。
- 当没有可计算结果时显示空状态。

## 成就榜

- 成就榜点数基于 `players[].collections` 计算。
- 每张图按鱼种顺序读取图鉴位图，若某一稀有度在该图内全部收集完成，则计入对应分值：N=1、R=2、SR=3、SSR=4、UR=5、UTR=10。
- 每张图的分值乘以地图 `id` 对应的数值；非数字 `id` 默认不计入基数，猫猫乐园 `S1` 例外使用基数 20。
- 成就榜排行仍按成就点数从高到低排序，展示时仅保留名次与成就点数。
- 成就榜浮窗中的地图方块会按 N 到 UR 的完成情况逐步填充颜色并提升高度，每完成一个稀有度增加 1/5，高度满格后若 N 到 UR 已全收集则在方块外侧显示皇冠；UTR 全收集不改变高度和皇冠条件，只将方块填充色切换为 UTR 颜色。

## 其他展示

- 选中地图时会显示该地图下各鱼种在不同稀有度下的单鱼价格提示表；若当前存在猫猫乐园建筑鱼价加成，提示表会显示 Alt 快捷键提示，按住 Alt 时临时显示原价，松开后恢复加成价；若已匹配玩家图鉴数据，玩家未收集的鱼会在对应价格处用稀有度颜色描边提示；提示表底部追加“成就”行，按当前稀有度下所有鱼的原始价格总和再乘以 3 计算，不计入猫猫乐园建筑鱼价加成，并用背景色和鱼种价格行区分。
- 页面标题与版本徽标由配置中的版本前缀和提交计数拼接生成。

## 交互要求

- 任一输入变化后，页面立即重算并刷新显示。
- 自动更新数据同步玩家鱼竿、鱼钩等级、药水和猫猫乐园建筑等级后，原生选择框、药水浮窗、建筑弹窗和页面展示值必须同步刷新。
- 开启鱼饵提醒或修改提醒阈值后，应立即按当前玩家鱼饵数量检查一次；页面从后台恢复可见时也应补充检查。
- 切换地图时刷新摘要、地图高亮和鱼饵表。
- 调整地图打窝 buff 时，仅影响对应地图的收益计算，不影响其他地图。

## 同步要求

后续任何涉及公式、鱼钩、鱼竿、地图、概率、buff、收益字段或展示逻辑的变更，都必须同步更新本文件。

## Cloudflare Worker sync

Nest buff data is served by the Cloudflare Worker in `workers/nest-buff-sync`.
Because the upstream IP blocks Cloudflare Worker egress, refresh is pushed by an
external sync runner instead of a Worker cron trigger. The Tencent Cloud
Function fetches the upstream JSON and writes it directly to Cloudflare KV with
the Cloudflare API token. The page reads `/nest-buff.json` through `config.js`
`nestBuffSourceUrl`.
